# 작업전 필수숙지 최상위 인덱스

역할: 사용자가 특정 파일 하나만 링크하거나 짧게 작업을 요구해도, 작업자는 이 문서를 먼저 열고 세계관/문체/작업규칙의 필수 층을 확인한 뒤 착수한다. 이 문서는 원고용 설정집이 아니라 작업 전 관문이다.

이 파일을 읽지 않고 세계관, 회차, 원고, 인물, 문체, 설정 문서를 수정하지 않는다.

이 폴더 전체가 지키는 두 가지 대원칙.

- 문서는 세계관과 규칙의 정본이지 창작의 틀이 아니다. 무엇이 사실이고 무엇이 금지인지는 문서가 정하지만, 장면을 어떤 순서로 어떻게 쓸지는 작업자의 판단에 맡긴다. 문서의 예시·동선·체크리스트는 참조이지 강제가 아니다.
- 같은 정보는 한 문서에만 산다. 다른 문서가 그 정보를 쓸 때는 내용을 복사하지 않고 정본으로 링크한다. 설정과 EP 내용은 언제든 바뀔 수 있으므로, 수정할 때는 정본 한 곳만 고치고 7절의 전파 절차로 나머지를 맞춘다.

---

## 0. 작업 전 절대 순서

작업자는 아래 순서를 지킨다.

1. 사용자가 링크한 파일을 바로 고치지 않는다.
2. 먼저 이 파일에서 작업 유형을 판정한다.
3. `필수 코어 6문서`를 읽는다.
4. 작업 유형별 추가 문서를 연다.
5. 링크된 대상 파일을 읽는다.
6. 충돌/레거시/설정 오염 가능성을 검색한다.
7. 필요한 경우에만 수정한다.
8. 수정 뒤에는 레거시 키워드, 깨진 참조, 공백 오류를 확인한다.

사용자가 "계획만"이라고 하면 1~6까지만 진행하고 파일을 고치지 않는다. 사용자가 "반영", "진행", "작성", "수정"을 요구하면 1~8까지 진행한다.

### 1.1 회차 집필 전달 패키지

`세계관 설정/` 폴더는 세계관·인물·규칙의 정본이지만, 회차 집필에는 폴더 밖 자료 두 가지가 함께 필요하다. 다른 작업자에게 회차를 맡길 때는 아래를 반드시 함께 전달한다.

| 동반 자료 | 위치 | 역할 |
|---|---|---|
| 회차 작업대 | `스토리/에피소드/해당 부/EP N.md` | 해당 회차의 중심 인물, 첫 장면, 체크리스트. `07`·`07G`의 해당 절과 링크로 연결되어 있다 |
| 직전 화 원고 | `스토리/웹소설작성/` | 연재 연속성의 기준. 문체 표본, 직전 사건, 말미 잔류물. 최소 직전 1~2화 |

세계관 설정 폴더만 전달하면 회차별 중심 인물과 직전 화 연결 지점에 도달할 수 없다.

소품은 상징을 위한 상징으로 쓰지 않는다. 사건을 실제로 바꾸거나 세계 법칙을 보여 줄 때만 전면화한다.

능력 설계 문서를 정리할 때는 아래 기준을 추가로 지킨다.

- `작가용 의미`, `핵심 질문`, `충돌 가치`는 삭제 대상이 아니다. 이것들은 연출과 복선 설계용 참조이며, 작중 대사나 전투 해설로 직접 노출하지 않는다.
- `전투 기능`은 먼저 한 컷으로 그릴 수 있어야 한다. 무엇이 나타나는지, 어디에 닿는지, 무엇을 막거나 뚫거나 늦추거나 바꾸는지, 상대가 어떻게 피하거나 반격할 수 있는지를 적는다.
- 능력이 `진실`, `논리`, `가치관`, `희망`, `기억`, `의지`, `선택`, `공리`, `전제` 같은 추상 개념 자체를 직접 공격하거나 고치는 식으로 쓰이면 안 된다. 그런 의미는 빛, 칼, 벌레, 씨앗, 거울, 실, 문서, 장치, 방벽, 통로, 손잡이처럼 보이거나 만질 수 있는 매개를 통해 간접적으로 드러낸다.
- 설명을 채우기 위해 뜻이 바로 잡히지 않는 조어를 만들지 않는다. 읽는 순간 장면이 그려지지 않는 표현은 금지어 목록으로 외우는 대신, 전투 중 보이는 소품, 동작, 실패 결과로 다시 쓴다.
- 지원형 능력은 허용한다. 다만 지원 방식도 방어, 이동, 회복, 감지, 교란, 증폭, 분산, 지연, 봉쇄, 전달, 구조 보조처럼 실제 장면에서 작동해야 한다.
- `능력 없음` 인물을 억지로 능력자로 만들지 않는다. 생활 행동, 판단, 소품 운용, 증언, 피난, 치료, 교섭, 기록, 현장 지원으로 장면 기능을 세운다.
- 사용자가 "무슨 의미냐", "이상하다", "전투 장면으로 안 보인다"고 지적하면 문장을 방어하지 말고, 먼저 실제 전투/지원 장면으로 구현 가능한지 감사한다.

## 3. 사용자 발화 우선 원칙

사용자의 채팅은 작업 요구사항의 1차 근거다. 작업자가 과거에 만든 요약, 임시 정리, 보조 문서는 사용자의 직접 발화를 대체하지 않는다.

| 사용자가 말한 것 | 처리 |
|---|---|
| "내 채팅만 확인해서" | 사용자 발화에서 확정/폐기/조건부 요구를 먼저 추출한다 |
| "레거시 남기지 말고" | 이전 설정을 후보, 주석, 보존용 문단으로 남기지 않는다 |
| "바로 수정하지 말고 계획만" | 수정 없이 계획, 충돌 후보, 검증 계획만 낸다 |
| "문체가 어색하다" | 한국어 자연스러움, 위치, 동선, 원인, 세계 법칙을 먼저 점검한다 |
| "무슨 의미냐" | 장치를 방어하지 말고 실제 문서와 세계 법칙상 필요한지 먼저 감사한다 |
| "웹검색/자료 활용" | 사용자가 명시한 자료나 최신 정보가 필요하면 먼저 확인하고, 그대로 베끼지 않고 기능만 재창조한다 |

## 4. 현재 확정된 핵심 세계관 잠금

이 절은 작업 중 가장 자주 오염되는 규칙을 잠그기 위한 압축판이다. 자세한 기준은 `01C`, `03B`, `03C`, `08`을 따른다.

### 4.1 낮과 밤

세부 규칙의 정본은 `01_세계관_상위규칙.md` 2A절과 `01C` 4A~4B다. 가장 자주 오염되는 지점만 잠근다.

- 밤은 발현이 있는 존재만 같은 좌표의 다른 위상으로 넘어가는 전환이다. 발현 없는 낮 주민은 하루 종일 낮에 남고, 하늘·빛·온도 변화를 감각하지 못한다.
- 밤 전환은 찰나에 일어난다. 석양 같은 중간 단계나 장기적 두근거림으로 쓰지 않는다.
- 낮 복귀 위치는 밤에서 마지막으로 있던 자리다. 막혀 있으면 가까운 빈자리로 밀려난다.
- 부서진 건축물과 큰 사물은 낮의 마지막 상태가 우선되고, 인물의 상처, 기억, 손에 쥔 작은 물건, 관계 불신은 이어질 수 있다.

### 4.2 호칭과 생활 인식

- 작중 생활어는 `밤의 사람`을 기본으로 한다.
- 초반 낮 주민은 달을 직접 본 적이 없으므로 `달을 본 사람` 같은 호칭을 쓰지 않는다.
- 낮 주민은 사라졌다 돌아온 사람들이 밤이나 달을 묘사했다는 소문 정도만 안다.
- 안대는 낮 주민의 평범한 수면 도구다. 밤을 막는 보호구가 아니다.
- 마감 종, 장터 정리, 문 닫기 같은 생활 신호는 밤 전환의 원인이나 방어책이 아니다.

### 4.3 밤의 위험

- 밤 자체가 무작위로 바닥을 가르거나 문이 박수를 치게 만들지 않는다.
- 첫 밤의 위험은 밤의 사람들에게 커지는 경쟁, 증명, 결투 충동과 발현자의 행동에서 생긴다.
- 정체 미상 존재는 정답을 직접 부르지 않는다. `포보스 악몽`, `포브스 악몽` 같은 명명은 원고에서 피한다.
- 위험은 관측 가능한 흔적으로 쓴다. 빈자리, 상처, 남은 손자국, 식은 금속 냄새, 놓친 소리, 끊어진 실처럼 보이거나 만질 수 있어야 한다.

### 4.4 셀로 초반 상태

- 셀로는 방금 태어난 존재가 아니다.
- 말, 걷기, 배고픔, 자기 이름, 밤 진입 감각을 알고 있다.
- 다만 자기애가 정식 발현되기 전이라 공허하고, 자기 정체보다 타인의 흔들림을 먼저 본다.
- 셀로는 밤 진입을 이상하게 여기지 않는다. 오히려 낮 사람들이 사라지는 현상에 궁금증을 가진다.
- 초반의 무지개빛 보호막은 보호이면서 동시에 고립이다. 셀로만 안전하게 만들고 막 밖의 손을 늦게 듣게 할 수 있다.

## 5. 원고 문체 잠금

문체 기준의 정본은 `08_원고용_작중어_빠른검수표.md`와 `08A_작중어_전용_설정집.md`, 일반 작법은 `08B_웹소설_문체_작법_기본.md`다. 여기서는 순서만 잠근다.

1. 먼저 그려지게 쓴다.
2. 그다음 낯설게 만든다.
3. 마지막에 생각하게 만든다.

첫 500자는 위치, 즉시 문제, 실패 비용을 포함한다. 아름다운 문장보다 먼저 독자가 머릿속에 장면을 그릴 수 있어야 한다. 상징을 위한 상징, 원인 없는 의인화, 번역체, 어색한 조어의 세부 금지 목록은 `08`과 아래 6절 검색표를 따른다. 외부 작품의 장점은 기능만 분석해 재창조하고, 장면·감정 표지·상징을 그대로 가져오지 않는다.

## 7. 작업 완료 전 검증

수정 뒤에는 최소한 아래를 확인한다.

1. 대상 파일이 작업 요구를 실제로 반영했는가.
2. `07`, `07G`, 해당 EP 문서, `08` 사이에 충돌이 남았는가.
3. 인물 대사/행동이 해당 회차 시점 정보만 사용했는가.
4. 레거시 키워드가 현재 작업 범위에 남았는가.
5. 새 문서를 불필요하게 만들지 않았는가.
6. `git diff --check`에서 공백 오류가 없는가.
7. 사용자가 커밋을 요구한 경우에만 선택적으로 스테이징/커밋했는가.

### 7.1 설정·EP 내용 변경 시 전파 절차

설정, 규칙, EP 내용은 확정본이 아니라 언제든 바뀔 수 있다. 바꿀 때 레거시가 남지 않도록 아래 순서를 지킨다.

1. 정본 한 곳만 고친다. 그 정보가 사는 정본 문서가 어디인지 먼저 확인하고, 다른 문서에 같은 내용이 복사되어 있으면 이번 기회에 링크로 바꾼다.
2. 옛 명칭과 옛 서술을 `스토리` 폴더 전체에서 검색한다. 세계관 설정만이 아니라 `에피소드`, `웹소설작성`, `장면시퀀스`까지 포함한다. 명칭 교체의 세부 절차는 `01E_명칭_능력_소품_작명기준.md` 9절(개명 위생)이 정본이다.
3. 문서 간 참조는 항상 상대경로 마크다운 링크로 쓴다. 경로 없는 파일명이나 저장소 루트 기준 경로를 쓰지 않는다. 파일을 옮기거나 이름을 바꿨으면 그 파일을 가리키던 링크를 전부 검색해 고친다.
4. 수정 범위의 링크가 실제 파일에 닿는지, `git diff --check`에 공백 오류가 없는지 확인하고 끝낸다.

옛 설정은 후보, 주석, 보존 문단으로 남기지 않는다. 계보 기록이 정말 필요한 경우에만 해당 정본 문서 안에 한 줄로 남긴다.

## 8. 최종 답변 기준

작업자는 최종 답변에서 아래를 짧게 말한다.

- 무엇을 바꿨는지.
- 어느 파일에 반영했는지.
- 어떤 검증을 했는지.
- 못 한 것이 있으면 왜 못 했는지.

장황한 세계관 설명보다 실제 반영 파일과 검증 결과를 우선한다. 사용자가 질문을 요구한 경우에는 수정 대신 질문으로 끝낸다.
