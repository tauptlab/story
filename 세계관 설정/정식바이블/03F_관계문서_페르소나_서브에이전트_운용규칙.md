# 관계문서와 페르소나 서브에이전트 운용규칙

역할: 특정 상대 앞에서만 달라지는 대사와 행동을 개별 인물 문서에 모두 넣지 않고, 관계 문서와 회차 제한 페르소나로 분리해 운용하는 기준이다.

---

## 1. 관계 문서를 만드는 기준

모든 관계를 문서화하지 않는다. 아래 중 하나라도 해당할 때만 관계 문서를 만든다.

| 기준 | 설명 |
|---|---|
| 말투 변화 | 특정 상대 앞에서 문장 길이, 존대, 농담, 침묵 방식이 달라진다 |
| 행동 변화 | 특정 상대 앞에서만 도망, 보호, 공격, 회피, 고백이 달라진다 |
| 숨기는 정보 | 서로에게 말하지 않는 과거, 빚, 두려움이 장면 선택을 바꾼다 |
| 결투 기능 | 두 사람의 가치관 충돌이 실제 전투나 선택을 만든다 |
| 장기 변화 | 관계가 여러 부에 걸쳐 변하고, 이후 대사를 계속 바꾼다 |

단순한 친분, 한 번의 만남, 배경 설명만 있는 관계는 개별 인물 문서의 관계 항목으로 충분하다.

관계 문서명은 기존 형식을 따른다.

```md
등장인물/관계- A, B.md
```

---

## 2. 관계 문서 형식

```md
# A와 B 관계 운용

정사 등급:
문서 유형: 관계 운용 문서

## 관계 핵심

## 서로에게 원하는 것

| 인물 | 원하는 것 | 숨기는 것 | 두려워하는 것 |
|---|---|---|---|

## 대사 변화

## 행동 변화

## 결투/갈등 운용

## 회차별 공개 정보

## 금지
```

관계 문서에는 두 사람의 전 생애를 반복하지 않는다. 각 인물의 절대 성격은 개별 인물 문서에 두고, 관계 문서에는 "이 둘이 함께 있을 때만 생기는 변형"을 둔다.

---

## 2-1. B급 이상 인물별 관계/페르소나 카드

B급 이상/강화 가능 인물은 장면 투입 빈도가 높으므로, 전용 1:1 관계 문서가 없더라도 인물별 관계/페르소나 카드를 먼저 둔다.

기본 위치:

```md
등장인물/관계운용/B등급이상/{인물}_관계페르소나.md
```

인물별 카드는 특정 쌍의 서사를 확정하는 문서가 아니다. 해당 인물이 관계 앞에서 어떻게 말투, 행동, 침묵, 거리, 밤 충동을 바꾸는지 정리하는 서브에이전트 입력 카드다. 특정 두 사람의 관계가 반복 장면, 결투, 숨기는 정보, 장기 변화를 만들면 그때 별도 `관계- A, B.md` 문서로 분리한다.

운용 순서:

1. 인물 문서의 `관계 문서` 줄에서 인물별 관계/페르소나 카드를 연다.
2. 현재 회차 사용 가능 범위와 밤 진입/발현 제한을 확인한다.
3. 상대가 전용 1:1 관계 문서에 걸리면 전용 문서를 우선한다.
4. 전용 문서가 없으면 인물별 카드와 `관계-B등급이상_운용색인.md`만으로 대사/행동 후보를 만든다.
5. 같은 조합이 반복되어 말투 변화나 결투 기능이 누적되면 별도 관계 문서를 만든다.

---

## 3. 페르소나 서브에이전트의 역할

페르소나 서브에이전트는 대사와 행동을 직접 제안하고, 자신이 제안한 대사와 행동을 다시 검수한다. 단, 해당 회차 시점까지의 정보만 볼 수 있다.

서브에이전트가 할 수 있는 일:

| 가능 | 설명 |
|---|---|
| 대사 생성 | 해당 인물이 지금 할 법한 문장을 2~5개 제안 |
| 행동 생성 | 손짓, 이동, 침묵, 공격, 회피 등 장면 행동 제안 |
| 자기 검수 | 이 인물답지 않은 문장, 설정 충돌, 말투 위반을 표시 |
| 관계 반응 | 특정 상대 앞에서 바뀌는 말투와 행동을 반영 |
| 지역성 반영 | 출신/거주/소속 지역의 사고 순서를 대사에 반영 |

서브에이전트가 하면 안 되는 일:

| 금지 | 이유 |
|---|---|
| 미래 결말 정보 사용 | 메인 에이전트가 복선과 장기 구조를 관리한다 |
| 새 능력 발명 | 능력 체계는 `03`과 개별 인물 문서가 우선한다 |
| 새 과거사 확정 | 인물 생애는 메인 에이전트 승인 후 문서화한다 |
| 세계 법칙 설명자화 | 대사는 인물의 현재 인식 안에 머물러야 한다 |
| 문체 검수 생략 | 생성 뒤 반드시 어색한 한국어와 번역체를 걸러야 한다 |

---

## 4. 서브에이전트 입력 패킷

장면마다 메인 에이전트는 아래 정보만 제공한다.

```md
## 현재 회차
- 부/EP/화:
- 장면 위치:
- 현재 시점까지 독자에게 공개된 정보:
- 이 인물이 알고 있는 정보:
- 이 인물이 모르는 정보:

## 참조 문서
- 인물 문서:
- 존재 위상:
- 애니어그램:
- 지역성:
- 인물별 관계/페르소나 카드:
- 관계 문서:
- 문체 규칙:

## 장면 목표
- 표면 목표:
- 숨은 감정:
- 반드시 남길 물성:
- 금지할 설명:
```

서브에이전트 입력 패킷에는 미래 복선, 결말부 연결, 세계관 장기 구조를 넣지 않는다. 이 정보는 메인 에이전트가 별도로 관리하고, 서브에이전트는 "지금 이 인물이 알 수 있는 것"만으로 움직여야 한다.

---

## 5. 서브에이전트 출력 형식

```md
## 현재 회차 인식
- 알고 있는 것:
- 모르는 것:
- 오해하고 있는 것:

## 장면 목표
- 원하는 것:
- 피하려는 것:
- 무의식적으로 드러나는 것:

## 행동 후보
1.
2.
3.

## 대사 후보
1.
2.
3.

## 자기 검수
- 이 인물답지 않은 행동:
- 말투 위반:
- 설정 충돌:
- 어색한 한국어:
- 더 나은 선택:
```

출력은 후보여야 한다. 최종 문장은 메인 에이전트가 회차 전체 리듬, 복선, 문체를 보고 고른다.

---

## 6. 메인 에이전트의 책임

메인 에이전트는 아래를 독점 관리한다.

| 책임 | 설명 |
|---|---|
| 결말과 처음의 연결 | 초반 장면이 최종부와 어떻게 공명하는지 관리한다 |
| 미래 복선 | 해당 회차 인물이 모르는 정보는 대사에 새지 않게 한다 |
| 세계관 법칙 | 낮/밤, 발현, 귀환, 결투 원리를 지킨다 |
| 문체 최종 선택 | 어색한 조어, 번역체, 과한 상징을 제거한다 |
| 상태 갱신 | 회차 뒤 바뀐 관계, 밤 진입, 발현 단계를 문서에 반영한다 |

서브에이전트는 인물의 현재 목소리를 만든다. 메인 에이전트는 작품 전체의 방향과 비밀을 지킨다.

---

## 7. 최종 검수 질문

장면을 확정하기 전 아래를 확인한다.

1. 이 대사는 현재 회차의 이 인물이 알 수 있는가.
2. 이 행동은 인물 문서의 절대 속성과 충돌하지 않는가.
3. 특정 상대 앞에서만 달라지는 반응을 관계 문서가 설명하는가.
4. 지역 말투가 읽기 어려운 방언으로 흐르지 않았는가.
5. 애니어그램이 라벨 설명이 아니라 불안과 선택으로 보이는가.
6. 밤의 사람이라면 밤 충동이 장면에 반영되었는가.
7. 낮의 사람이라면 밤을 직접 아는 듯 말하지 않았는가.

이 일곱 질문을 통과하지 못한 대사는 아무리 멋있어도 버린다.
